## Cambios necesarios en el código

### 1. Archivos JSON — agregar `route3`

Cada archivo (`morning.json`, `midday.json`, `evening.json`) debe incluir una clave `route3` al mismo nivel que `route1` y `route2`:

```json
{
  "route1": [ ... ],
  "route2": [ ... ],
  "route3": [
    {
      "id": "3A",
      "name": "Ruta 3A",
      "direction": "Ida",
      "start": "Garaje Transportes",
      "end": "FCS",
      "schedule": "7:05 AM - 7:30 AM",
      "startTime": "7:05",
      "color": "#2ca02c",
      "points": [ ... ],
      "streets": [ ... ]
    }
  ]
}
```

### 2. `js/main.js` — función `loadAllRoutes`

Agregar `route3` a la carga:

```js
if (routesData.route3) {
  routesData.route3.forEach((r) => (r.period = period));
  allLoadedRoutes.push(...routesData.route3);
}
```

### 3. `js/main.js` — función `showRoutes`

Agregar soporte para `routeType === 'route3'` y actualizar el caso `'both'` para incluir las 3 rutas:

```js
} else if (routeType === "route3") {
  routesToShow = routesData.route3 || [];
}
```

Y en el caso `"both"`, cambiar el filtro para incluir también los tramos de Ruta 3.

### 4. `index.html` — menú lateral

Agregar botón "Ruta 3" en cada sección del menú:

```html
<button class="btn btn-route3" onclick="showRoutes('morning', 'route3')">🚌 Ruta 3</button>
```

Y su estilo (puede usar el color verde `#2ca02c` o uno nuevo):

```css
.btn-route3 { background: #16a34a; }
```

## Formato de coordenadas

geojson.io exporta `[longitud, latitud]`. El proyecto usa `[latitud, longitud]`. Invertir al convertir.

**GeoJSON exportado:**
```json
"coordinates": [[-67.065, -17.976], [-67.068, -17.975]]
```

**Formato para el JSON del proyecto:**
```json
"points": [[-17.976, -67.065], [-17.975, -67.068]]
```
